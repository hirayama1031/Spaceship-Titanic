# Spaceship Titanic: 乗客輸送予測プロジェクト

このプロジェクトは、宇宙船「Spaceship Titanic」の事故において、どの乗客が異次元に輸送（Transported）されたかを予測する分類タスクです。データ前処理から複数の機械学習モデルの比較、ハイパーパラメータ最適化までを段階的に実施しました。

## 1. データ前処理と特徴量エンジニアリング

精度の向上に最も貢献したのは、単なる欠損値補完ではなく、「グループ（家族・友人）」の概念を取り入れた以下の手法です。

- **グループベースの補完**: `HomePlanet` や `Cabin` などの欠損値を、同じグループ ID を持つ仲間の最頻値で補完。同じ家族は同じ場所から出発し、同じ区画に泊まるという仮説をデータに反映。
- **特徴量の分割**: `Cabin` 列（Deck/Num/Side）を 3 つの独立した特徴量に分割。特に `Side`（左舷・右舷）と `Deck`（階層）が予測に有効であることを確認。
- **支出合計の算出**: `RoomService` や `Spa` などの各支出項目を合算した `TotalExp` を作成。
- **家族構成の抽出**: 名前から名字（Surname）を抽出し、家族人数（FamilySize）を算出。

## 2. モデル別性能比較

4つの異なるアルゴリズムを実装し、その特性を比較しました。

| モデル | 検証データ正解率 (Accuracy) | 特徴 |
| :--- | :--- | :--- |
| **ロジスティック回帰** | 0.7900 | シンプルで解釈性が高い。Europa 出身や CryoSleep 中の人が転送されやすいという「全体傾向」を捉えた。 |
| **決定木 (深さ5)** | 0.7878 | 判断プロセスを可視化。支出合計（TotalExp）の有無が最も重要な分岐点であることを発見。 |
| **ランダムフォレスト** | 0.8016 | 100本の木の多数決。`Cabin_num` や `Age` といった連続的な数値の重要度が上昇し、精度が向上。 |
| **LightGBM + Optuna** | **0.8246** | **最強モデル**。Optuna による自動最適化により、複雑なパターン（L1/L2正則化、葉の数など）を学習。 |

## 3. LightGBM による重要度分析

最終モデル（LightGBM）において、予測に最も寄与した項目は以下の通りです。

1. **Cabin_num (客室番号)**: 船内の非常に細かい位置関係が転送に直結していた。
2. **Age (年齢)**: 特定の年齢層（特に子供）の保護状況などが影響。
3. **TotalExp (支出合計)**: 経済状況や、睡眠中（支出ゼロ）か活動中かの状態。
4. **FamilySize (家族人数)**: 家族で共に行動していたかどうかの指標。

## 4. 結論

本プロジェクトを通じて、単純な線形モデル（ロジスティック回帰）から高度な勾配ブースティング（LightGBM）へステップアップすることで、精度を 0.79 から 0.82 まで引き上げることができました。

特に、**「Cabin_num（部屋番号）」が最重要**という結果は、事故の被害が船内の特定の区画に極めて局所的に発生したことを示唆しています。ロジスティック回帰が見逃していた「部屋番号ごとの微細な生存率の差」を、LightGBM が詳細に捉えたことが高精度の要因となりました。
